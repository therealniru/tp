Edit Candidate Details (edit)

Allow a recruiter to update a candidate’s stored contact information
when new or corrected details are obtained.

Command Format: edit INDEX [n/NAME] [p/PHONE] [e/EMAIL]
(At least one field must be specified).

Parameters & Validation:
INDEX: Must be a positive integer corresponding to a valid index
in the currently displayed list. No leading zeros.
Trimmed of whitespace.

NAME (n/): 1-100 chars. Allowed: letters (A-Z, a-z), spaces,
hyphens (-), apostrophes ('), periods (.), and slashes (/).
Case-sensitive for storage, case-insensitive for duplicate
checking. Collapse multiple consecutive spaces into a single
space during storage. Trim whitespace from start/end.

PHONE (p/): 3-15 chars. Allowed: digits (0-9). Optional
'+' allowed ONLY as the first character. Trim whitespace.

EMAIL (e/): Max length 254 chars. Standard AB3 format.
Case-insensitive for storage and duplicate checking.
Trim whitespace.
